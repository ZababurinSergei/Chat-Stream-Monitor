## Тестирование

---

### Для UI-компонентов:

- **Приоритет 1: БАЗОВАЯ ФУНКЦИОНАЛЬНОСТЬ**  
  1. Пропсы — типы, значения по умолчанию, реактивность  
  2. События — emits, аргументы событий  
  3. Рендеринг — компонент вообще отображается  
- **Приоритет 2: КОНТЕНТ И ИНТЕРАКТИВНОСТЬ**  
  4. Слоты — default, named, scoped slots  
  5. Состояния — disabled, loading, error состояния  
  6. Пользовательский ввод — клики, ввод текста  
- **Приоритет 3: СТИЛИ И ИНТЕГРАЦИЯ**  
  7. CSS-классы — добавление/удаление по условиям  
  8. Интеграция с naive-ui — проброс пропсов, события  
  9. Кастомизация — через props/slots/css vars  
- **Приоритет 4: ПРОДВИНУТОЕ**  
  10. Доступность (a11y) — ARIA, фокус, клавиатура  
  11. Производительность — memoization, лишние рендеры  
  12. Edge cases — граничные случаи, ошибки  

### Для composables:

1. Возвращаемые значения — структура, реактивность
2. Параметры функции — аргументы, значения по умолчанию
3. Побочные эффекты — вызовы API, таймеры, события
4. Состояния — изменения состояния, реактивность
5. Очистка — cleanup функции, таймеры


## Дополнительные принятые правила

---

1. Для функций пишем JSDoc-документацию с параметрами и возвращаемым значением
2. Обязательно пишем интерфейсы для всего, что можно описать
3. Используем any очень осторожно! Проект не должен превратиться в кучу всего, у которого тип данных any
